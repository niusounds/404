# 怖い話ブログ執筆ガイドライン

このファイルは、AIアシスタントがこのリポジトリで怖い話を執筆する際のガイドラインです。

## 1. 基本的なフォーマット

### ファイル配置
- 記事は `_posts/` ディレクトリに配置
- ファイル名は `YYYY-MM-DD-kebab-case-title.md` 形式
- 画像は `assets/images/` ディレクトリに配置

### フロントマター
```yaml
---
title: 日本語のホラータイトル
---
```

## 2. 文章フォーマットの注意

- 原稿には　(全角スペース)で段落の最初がインデントされていますが、削除してください。インデントはスタイルシートで付けます。
- 画像を入れた後は改行を2つ入れてください。1つしか入れていないとHTMLにレンダリングされたときに改行にならないためです。
- 段落は短く。一文一段落も効果的に使う。

## 3. 画像（SVG）の作成

### 配置例
```markdown
![画像の説明](/assets/images/filename.svg)

*イタリックでキャプションを追加*
```

### SVGデザインのポイント
- **色調**: 暗い背景 (`#0a0808`, `#05050a`) にアクセントカラー
- **アニメーション**: `<animate>` で微妙な動きを追加（点滅、揺れ、明滅）
- **雰囲気**: 不安感を煽る要素（影、歪み、ぼかし）

## 4. 物語のジャンル

既存の話のジャンルを把握し、**新しいジャンル**に挑戦することを推奨します。

### 使用済みジャンル（2026年3月時点）
- テクノロジーホラー（スマホ、AI、スマートスピーカー、ビデオ会議）
- 入れ替わり/乗っ取り
- 異空間（存在しない階、終わりのない電車）
- 学校ホラー（修学旅行、教科書）
- 日常の違和感（鏡、窓、隣人）
- 言霊/名前ホラー

### 未開拓ジャンルの例
- 自然・土着ホラー（山、海、森、古い村）
- 食べ物ホラー
- 音・音楽ホラー
- 時間の歪みホラー
- 職人・伝統工芸ホラー
- 夢と現実の境界

## 5. 文体・トーンのガイドライン

- **視点**: 一人称（「私」「僕」）で没入感を高める
- **導入**: 日常的な場面から始め、徐々に違和感を忍び込ませる
- **クライマックス**: 心理的恐怖を重視。ジャンプスケアより不気味さ
- **結末**: ハッピーエンドは避ける。余韻を残す終わり方

## 6. クレジット

記事の最後に必ず以下の形式でクレジットを追加:

```markdown
---
Written by {モデル名}
```

例:
- `Written by GitHub Copilot (Claude Opus 4)`
- `Written by GitHub Copilot using Gemini 3 Flash (Preview)`

## 7. ビルド確認

変更後は以下のコマンドでJekyllビルドを確認:

```bash
bundle install    # 初回のみ
bundle exec jekyll build
```
