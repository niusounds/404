# 怖い話ブログ執筆ガイドライン

このファイルは、AIアシスタントがこのリポジトリで怖い話を執筆する際のガイドラインです。

## 1. 基本的なフォーマット

### ファイル配置
- 記事は `_posts/` ディレクトリに配置
- ファイル名は `YYYY-MM-DD-kebab-case-title.md` 形式
- 画像は `assets/images/` ディレクトリに配置

### フロントマター
```yaml
---
title: 日本語のホラータイトル
---
```

## 2. 文章フォーマットの注意

- 原稿には　(全角スペース)で段落の最初がインデントされていますが、削除してください。インデントはスタイルシートで付けます。
- 画像を入れた後は改行を2つ入れてください。1つしか入れていないとHTMLにレンダリングされたときに改行にならないためです。
- 段落は短く。一文一段落も効果的に使う。

## 3. 画像（SVG）の作成

### 配置例
```markdown
![画像の説明](/assets/images/filename.svg)

*イタリックでキャプションを追加*
```

### SVGデザインのポイント
- **色調**: 暗い背景 (`#0a0808`, `#05050a`) にアクセントカラー
- **アニメーション**: `<animate>` で微妙な動きを追加（点滅、揺れ、明滅）
- **雰囲気**: 不安感を煽る要素（影、歪み、ぼかし）

## 4. 物語のジャンル

怖い話は「お化け・怪異」だけではありません。**ミステリー**や**ヒトコワ**など、人間の暗部に迫る物語も積極的に書いてください。既存の話のジャンルを把握し、**新しいジャンル**に挑戦することを推奨します。

### 大分類

#### 怪異・超常ホラー
幽霊、妖怪、呪い、異界など超自然的な恐怖。

#### ミステリー
謎、事件、真相解明を軸にした物語。必ずしも超常現象は必要なく、「説明のつかない謎」や「真相がより恐ろしい」構造が読者を引き込む。
- 失踪、不可解な死、奇妙な遺留品
- 「なぜそこにいたのか」「なぜそうなったのか」が最後まで明かされない
- 調査・推理の過程で違和感が積み重なる

#### ヒトコワ（人間怖い）
幽霊より人間の方が怖い、という感覚を描く。超常現象ではなく、人間の狂気・執着・悪意・無関心が恐怖の源泉。
- ストーカー、隣人の異常行動、カルト・洗脳
- 笑顔の裏に潜む悪意、家族や友人が「別人」になっていく過程
- 「助けを求めても誰も信じてくれない」孤立感
- 犯罪・監視・支配といった社会的恐怖

### 使用済みジャンル（2026年3月時点）
- テクノロジーホラー（スマホ、AI、スマートスピーカー、ビデオ会議）
- 入れ替わり/乗っ取り
- 異空間（存在しない階、終わりのない電車）
- 学校ホラー（修学旅行、教科書）
- 日常の違和感（鏡、窓、隣人）
- 言霊/名前ホラー
- 自然・土着ホラー（山呼び）
- 食べ物ホラー（出汁の味、漬物と名前）
- 音・音楽ホラー（禁じられた民謡、444の鐘）
- 時間の歪みホラー（七分時計）
- 職人・伝統工芸ホラー（漆の肖像）
- 夢と現実の境界

### 未開拓テーマの例
- **ミステリー**: 不可解な失踪事件、謎の遺留品、解読できない暗号文、信頼できない証人
- **ヒトコワ**: ストーカー、SNSでの監視・つきまとい、カルト宗教への勧誘、職場での精神的支配
- **怪異**: 海・水辺の怪異、老人ホーム・介護施設での出来事、旅行先の土着信仰
- **その他**: 職場・会社ホラー、老いと記憶の喪失、異文化での恐怖体験

## 5. 多様化運用ルール（強め）

- 執筆前に直近3本を確認し、**ジャンル・構成・語り手**の重複を避ける。
- 直近3本と同系統のテーマは原則禁止。やむを得ず使う場合は、恐怖の核（謎、動機、結末）を別系統へ切り替える。
- テーマ選定は「未開拓テーマの例」を最優先。特にミステリー要素を意識して、早い段階で読者に問いを提示する。
- 同一の導入パターン（深夜帰宅中に異変、鏡を見ると違和感など）を連投しない。
- 同一のオチ類型（入れ替わり確定、ループ確定、語り手消失）を連続使用しない。

### 構成バリエーション規約
- 時系列順だけでなく、時系列逆転（結末断片から開始）を使う。
- 証言記録形式（供述、通話ログ、聞き書き）を定期的に採用する。
- 調査ログ形式（メモ、現地記録、回収資料）で手がかりを段階的に開示する。
- 断章形式（短い場面を複数連結）で、読者が後から全体像を復元する設計も許可する。

### 語り手バリエーション規約
- 一人称を基本としつつ、信頼できない語り手（記憶欠損、改ざん疑惑）を積極的に使う。
- 複数証言型（一人称×複数人）で、証言同士の矛盾を恐怖に変換する。
- 第三者記録型（編集者注、回収文書、監視記録）を織り交ぜ、読者の解釈余地を残す。

## 6. 文体・トーンのガイドライン

- **視点**: 一人称（「私」「僕」）で没入感を高める
- **導入**: 日常的な場面から始め、徐々に違和感を忍び込ませる
- **クライマックス**: 心理的恐怖を重視。ジャンプスケアより不気味さ
- **結末**: ハッピーエンドは避ける。余韻を残す終わり方

### ジャンル別のトーン調整

#### ミステリーを書く場合
- 謎を早めに提示し、読者を「なぜ？」という問いで引っ張る
- 手がかりを少しずつ散りばめ、解明の過程でより深い恐怖が露わになる構造にする
- 真相は「説明できない」か「説明できるが、それがより恐ろしい」で締める
- 語り手の信頼性を揺さぶる（信頼できない語り手）も有効

#### ヒトコワを書く場合
- 最初は「変わった人」程度の違和感から始め、徐々にその異常さを露わにする
- 「普通に見える」外見と「内側の狂気」のギャップを丁寧に描く
- 恐怖の根拠を現実的に保つ（超常現象に頼らない）
- 主人公の孤立・誰にも信じてもらえない状況を描くと効果的
- 読後に「これは現実にあり得る」と感じさせることが目標

## 7. クレジット

記事の最後に必ず以下の形式でクレジットを追加:

```markdown
---
Written by {モデル名}
```

例:
- `Written by GitHub Copilot (Claude Opus 4)`
- `Written by GitHub Copilot using Gemini 3 Flash (Preview)`
