# 404：存在しないはずの記憶

https://story.niusounds.com/

生成AIに文章を書かせたらどんな怖い話が出てくるんだろう？という興味から作ったWebサイトです。

文章以外にも様々な恐怖コンテンツをAIに企画してもらい、AIに実装してもらっています。

基本的に私は口出しせず、「いいですね、実装してください」「もっと怖くしてください」「恐怖要素を10個考え、実装してください」「新しいホラーコンテンツを考えて」みたいなプロンプトしか与えてないです。成果物もほとんどレビューせず、むしろ「もっと怖くして」とか言ってるだけ。

Webサイトを初期構築する段階では「フィルムグレインエフェクトを追加して」「怖いフォントにして」「時々文字が動く演出を付けて」「挿絵を付けて」とか言いましたが、ある程度形ができてきた後は「新しい記事を書いて」というだけで勝手に怖い話が量産されるようになりました。

AIが勝手に怖い文章を生成していく、ということ自体がホラーな気もします…という、サイトです。

このサイトを見たからと言って、呪われたりPCが壊れたりすることはないと思いますが、閲覧は自己責任でお願いします。

## 💀こだわりポイント

- **ノイズ（Film Grain）**: 視界を遮る微細な粒子状のノイズ。
- **グリッチ（Glitch Effect）**: 記憶の歪みを再現する画面の乱れ。
- **痙攣（Text Jerking）**: 読んでいる最中に微妙に震え、焦点を狂わせる文章。
- **フリッカー（Flicker）**: 常に明滅し、不安を煽る画面。

## 🛠 技術構成

- **Framework**: [Jekyll](https://jekyllrb.com/) 4.3+
- **Languages**: HTML5, Liquid, CSS3, JavaScript
- **Typography**: [New Tegomin](https://fonts.google.com/specimen/New+Tegomin), [Yuji Syuku](https://fonts.google.com/specimen/Yuji+Syuku) (from Google Fonts)
- **Styling**: フレームワークに頼らない、フルカスタムのCSSアニメーションとデザイン。

## 🚀 開発・動作環境

ローカル環境でサイトを起動するには、以下の手順に従ってください。

### セットアップ

https://jekyllrb-ja.github.io/docs/ に従い、Jekyllをインストールします。

1. 必要パッケージのインストール:
   ```bash
   bundle install
   ```

2. 開発用サーバーの起動:
   ```bash
   bundle exec jekyll serve --livereload
   ```

3. ブラウザで閲覧: `http://localhost:4000`

## 🌑 プロジェクト構造

- `_posts/`: 怪異の報告書（Markdown）
- `assets/css/style.css`: サイトの不気味な雰囲気を司るコアなデザインコード。
- `_layouts/`: 各ページの骨格。
- `index.html`: 記憶の集積所への入り口。
